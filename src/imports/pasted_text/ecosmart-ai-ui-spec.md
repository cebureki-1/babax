Ок, даю тебе чёткие требования для UI проекта EcoSmart AI — как для реального продукта + чтобы можно было показать преподавателю или добавить в портфолио.

🎯 1. Общая цель UI

UI должен:

показывать прогноз энергопотребления
отображать аналитику
давать рекомендации
быть простым и понятным
🧱 2. Структура интерфейса (страницы)
🟢 1. Dashboard (главная страница)

👉 Основной экран

Должно быть:

🔢 Текущий прогноз

Energy Consumption: 120 kWh
📈 График потребления
по часам / дням
⚠️ Статус
Normal / High / Critical
🟢 2. Analysis Page

👉 Более детальная аналитика

Должно быть:
📊 Графики:
Temperature vs Energy
Occupancy vs Energy
📉 История потребления
🟢 3. Recommendations Page

👉 Самая “AI часть”

Должно быть:

📌 Список рекомендаций:

- Turn off lights in empty rooms
- Reduce HVAC usage

🧠 Причина:

Because occupancy = 0
🟢 4. Settings (опционально)
выбор модели
загрузка CSV
переключение режима
🎨 3. UI компоненты (обязательно)
🔹 Cards (карточки)
Consumption
Temperature
Occupancy
🔹 Chart (график)

👉 важно:

line chart (основной)
bar chart (опционально)
🔹 Button
[ Analyze ]

👉 при нажатии:

отправка запроса на backend
🔹 Status Indicator

цвета:

🟢 green → нормально
🟡 yellow → повышено
🔴 red → критично
🧩 4. Минимальный layout
----------------------------------
| Sidebar |   Dashboard           |
|         |  [Cards]              |
|         |  [Chart]              |
|         |  [Recommendations]    |
----------------------------------
⚙️ 5. Логика UI (очень важно)
При загрузке:
запрос к API → получить данные
При нажатии Analyze:
axios.post('/predict')
Ответ:
{
  "prediction": 120,
  "status": "High",
  "recommendations": [
    "Turn off lights",
    "Reduce HVAC"
  ]
}
UI обновляет:
число
график
список рекомендаций
🎯 6. UX требования
не больше 2–3 кликов до результата
понятные подписи
не перегружать графиками
всё читаемо
🧪 7. Что обязательно показать на защите

👉 Минимум:

✅ прогноз (число)
✅ график
✅ рекомендации
✅ кнопка Analyze
